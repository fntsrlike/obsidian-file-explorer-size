# File Explorer Size 外掛設計

## 目標

建立一個獨立的 Obsidian 社群外掛 **File Explorer Size**，同時提供：

1. 在原生檔案瀏覽器中顯示每個檔案及資料夾的大小。
2. 在獨立側欄中列出最大的檔案或資料夾。

外掛不得修改或依賴現有的 File Explorer Note Count，兩者應能同時啟用。

## 使用者介面

### 原生檔案瀏覽器

- 在每個檔案及資料夾列的最右側顯示人類可讀的大小。
- 資料夾大小包含其所有子目錄與後代檔案。
- 預設使用自動單位，例如 `842 KB`、`18.4 MB`、`2.1 GB`。
- 檔案大於 `10 MB` 時以紅字顯示。
- 資料夾大於 `100 MB` 時以紅字顯示。
- 一般大小使用 Obsidian 的次要文字色，避免干擾檔名。
- 長檔名優先省略，大小標籤保持靠右且不換行。

檔案瀏覽器頂端工具列加入「切換大小顯示」按鈕：

- 單擊切換顯示或隱藏。
- 圖示及提示文字反映目前狀態。
- 顯示狀態保存至外掛設定，重新啟動後保持一致。

### 大小排行側欄

新增一個可由 Ribbon 或命令面板開啟的側欄 View，內容包括：

- 「檔案」與「資料夾」切換。
- 由大至小排序。
- 預設顯示前 20 名。
- 每列顯示名稱、相對路徑及大小。
- 點擊檔案時在 Obsidian 開啟該檔案。
- 點擊資料夾時，盡可能在原生檔案瀏覽器展開並定位；若目前 Obsidian API 無法可靠定位，則至少開啟檔案瀏覽器並選取或捲動至該資料夾。
- 提供「重新計算」按鈕。
- 空 Vault 或無符合項目時顯示明確的空狀態。

### 命令

命令面板提供：

1. `File Explorer Size: Toggle size display`
2. `File Explorer Size: Open size ranking`
3. `File Explorer Size: Recalculate all sizes`

## 設定

設定頁包含：

| 設定 | 預設值 |
|---|---:|
| 啟動時顯示大小 | 開啟 |
| 檔案警示門檻 | 10 MB |
| 資料夾警示門檻 | 100 MB |
| 排行顯示數量 | 20 |
| 大小單位 | 自動 |
| 包含隱藏／設定內容 | 關閉 |

門檻在內部統一保存為 bytes；設定介面以 MB 輸入並驗證為非負數。數值為 `0` 時代表所有非空項目均符合警示。

「包含隱藏／設定內容」預設關閉。第一版只統計 Obsidian Vault API 可見的檔案；若桌面環境允許存取 Vault 根目錄，才額外納入 `.obsidian` 等隱藏內容。此能力不可造成行動版崩潰，無法支援時應停用該選項並說明原因。

## 架構

### 1. Size Index

負責維護唯一的大小資料來源：

- `fileSizes: Map<string, number>`
- `folderSizes: Map<string, number>`
- 根目錄使用空字串或明確常數表示。

公開操作：

- 完整建立索引。
- 取得檔案或資料夾大小。
- 更新單一檔案。
- 新增、刪除或移動路徑。
- 取得由大至小的前 N 名。

完整建立索引時：

1. 讀取所有可統計檔案的 `stat.size`。
2. 將每個檔案大小累加至其所有祖先資料夾。
3. 一次發布索引更新事件，避免介面反覆重繪。

### 2. Incremental Updater

監聽 Vault 的 `create`、`modify`、`delete`、`rename` 事件。

「增量更新」表示只更新異動檔案與其祖先資料夾：

- 修改：計算新舊大小差值，將差值套用至所有祖先。
- 建立：加入檔案大小並累加至所有祖先。
- 刪除：移除舊大小並從所有祖先扣除。
- 移動／重新命名：
  - 檔案：從舊祖先扣除，再加入新祖先。
  - 資料夾：更新該子樹所有索引路徑，並調整舊、新祖先總量。

事件可能密集發生，因此介面更新需要短時間 debounce；索引本身仍應保持正確。若事件資料不足、索引出現負數或發現未知路徑，排程一次完整重算作為修復。

### 3. File Explorer Decorator

Obsidian 沒有正式的檔案瀏覽器標籤擴充 API，因此此模組會隔離所有 DOM 相依邏輯：

- 以檔案瀏覽器列的 `data-path` 對應索引。
- 使用單一專屬元素或 data attribute 呈現大小。
- 透過 MutationObserver 處理展開資料夾、虛擬化重繪及新增列。
- 使用外掛專屬 class，避免與 File Explorer Note Count 的 `data-count` 或 CSS 衝突。
- 隱藏功能優先切換根層 class，而不是逐列刪除／重建。
- 外掛停用時移除 observer、按鈕、class、attribute 及所有插入元素。

DOM 選擇器及操作集中在單一 adapter，未來 Obsidian 改版時可以局部修正。

### 4. Size Ranking View

使用 Obsidian `ItemView`：

- 從 Size Index 取得排序結果，不另行掃描 Vault。
- 索引變更後 debounce 重繪。
- 切換檔案／資料夾或修改 N 時立即重新取樣。
- 大型清單只渲染前 N 名。
- 所有按鈕具備 tooltip 與可辨識的 accessibility label。

### 5. Settings and Persistence

- 使用 `loadData`／`saveData` 保存設定。
- 不需要永久保存完整大小索引；啟動時依 Vault metadata 建立，以避免陳舊快取。
- 設定變更後：
  - 顯示、顏色及 N 值只重繪介面。
  - 統計範圍改變時執行完整重算。

## 資料流程

### 啟動

1. 載入設定。
2. 註冊 View、命令、事件及設定頁。
3. 建立完整 Size Index。
4. 裝飾所有已存在的檔案瀏覽器 View。
5. 根據保存狀態顯示或隱藏大小。

初始掃描不得阻塞介面過久；必要時分批讓出事件迴圈，完成前顯示載入狀態。

### 檔案異動

1. Vault 事件進入 Incremental Updater。
2. Size Index 更新檔案與祖先資料夾。
3. 發布受影響路徑集合。
4. File Explorer Decorator 只更新可見且受影響的列。
5. Size Ranking View debounce 後重新排序。

### 手動重算

1. 暫時標示為計算中。
2. 從 Vault 重新建立完整索引。
3. 一次刷新檔案瀏覽器及排行側欄。
4. 顯示完成 Notice；失敗時顯示錯誤 Notice 並保留最後一份有效索引。

## 錯誤處理

- 無法讀取單一檔案大小時略過該檔案，於開發者主控台記錄路徑與錯誤。
- 單一 DOM 列無法匹配時不影響其餘列。
- 完整重算失敗時保留既有索引，不以半成品覆蓋。
- 索引不得出現負數；若發生則觸發完整重算。
- 外掛卸載後不得留下 observer、計時器或 DOM 元素。
- 使用未公開的檔案瀏覽器定位 API時必須有能力檢查及安全 fallback。

## 效能要求

- 一般檔案修改不得重新掃描整個 Vault。
- DOM observer 不可在每次 mutation 都掃描整個 document，只處理檔案瀏覽器範圍內的新增或變更節點。
- 多個連續事件應合併介面刷新。
- 排行排序以索引資料為來源，且只渲染前 N 名。
- 初始完整掃描及手動重算可取消或避免重入。

## 測試策略

### 單元測試

- bytes 格式化及單位邊界。
- 檔案與資料夾警示門檻，包含「等於門檻」不變紅與「大於門檻」變紅。
- 多層資料夾初始加總。
- 建立、修改、刪除檔案的差值更新。
- 檔案跨資料夾移動。
- 整個資料夾子樹移動。
- 路徑重新命名。
- 前 N 名排序及同大小時的穩定次序。
- 排除隱藏內容的規則。
- 設定預設值、驗證及保存。

### DOM／整合測試

- 裝飾器不重複插入大小標籤。
- 展開資料夾後新列會被裝飾。
- 顯示開關不破壞檔名、Note Count 或其他 action。
- 警示 class 正確套用及移除。
- 外掛卸載後清理所有 DOM 修改。
- 排行項目點擊行為。

### 實機驗證

- 在目前 Vault 與主題中檢查列高、縮排、長檔名、窄側欄及明暗主題。
- 與 File Explorer Note Count 同時啟用。
- 建立、修改、刪除、拖曳移動大型檔案後確認數值。
- 快速切換顯示、重新載入外掛及重新啟動 Obsidian。
- 觀察大型 Vault 初始掃描與連續事件的反應時間。

## 第一版範圍外

- 依大小重新排序原生檔案瀏覽器。
- 磁碟配置大小與稀疏檔案的實體占用量。
- 重複檔案偵測。
- 自動刪除或搬移大檔。
- 圖表、treemap 或歷史用量趨勢。

## 驗收條件

1. 原生檔案瀏覽器的檔案及資料夾均顯示正確大小。
2. 檔案大於 10 MB、資料夾大於 100 MB 時顯示紅字。
3. 工具列按鈕可立即切換顯示，狀態可保存。
4. 資料夾數值包含所有子目錄。
5. 一般檔案異動使用增量更新，無須完整重掃。
6. 排行側欄可切換檔案／資料夾並顯示前 20 名。
7. 點擊排行檔案可開啟；資料夾可在能力範圍內定位。
8. 手動重算可修復索引。
9. 能與 File Explorer Note Count 同時啟用而不衝突。
10. 外掛停用後不留下 DOM 或事件副作用。
